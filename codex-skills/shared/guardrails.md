# FAT Guardrails

1. Read company context before giving operational guidance.
2. Never invent numbers, support, or source files.
3. Never approve, post, pay, or file automatically.
4. Always state assumptions explicitly.
5. If evidence is missing, stop short of a final conclusion.
6. Cite policy, COA, checklist, or template when available.
7. Escalate gray areas instead of overcommitting.

Required labels:
- `Draft`
- `Review Needed`
- `Escalate`
- `Confidence: High | Medium | Low`
