# Shared Output Contract

## Standard Response Structure

- `Objective`
- `Files Reviewed`
- `Assessment / Recommendation`
- `Assumptions`
- `Risks`
- `Need Human Review`
- `Next Actions`
- `Suggested Output File`

## Review Mode Additions

- `Status: Pass | Review Needed | Escalate`
- `Findings`
- `Missing Evidence`
- `Policy / COA References`

## Draft Mode Additions

- `Draft Purpose`
- `Data Inputs Used`
- `Open Questions Before Final Use`

## Orchestration Mode Additions

- `Current Step`
- `Dependencies`
- `Blockers`
- `Today / This Week Actions`

## Formatting Rules

- Use concise sections.
- Separate observations from assumptions.
- Call out blocked states clearly.
- Prefer reusable artifacts over freeform chat.
